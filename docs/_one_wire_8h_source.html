19// You can exclude certain features from OneWire. In theory, this
20// might save some space. In practice, the compiler automatically
21// removes unused code (technically, the linker, using -fdata-sections
22// and -ffunction-sections when compiling, and Wl,--gc-sections
23// when linking), so most of these will not result in any code size
24// reduction. Well, unless you try to use the missing features
25// and redesign your program to not need them! ONEWIRE_CRC8_TABLE
26// is the exception, because it selects a fast but large algorithm
27// or a small but slow algorithm.

39// Select the table-lookup method of computing the 8-bit CRC
40// by setting this to 1. The lookup table enlarges code size by
41// about 250 bytes. It does NOT consume RAM (but did in very
42// old versions of OneWire). If you disable this, a slower
43// but very compact algorithm is used.
